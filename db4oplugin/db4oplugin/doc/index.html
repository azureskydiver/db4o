db4o Instrumentation

This plugin adds support for bytecode instrumentation for db4o Transparent Activation/Transparent Persistence (TA/TP) within the Eclipse IDE. It provides a builder that will react to changes to class files (most likely triggered by the Java builder) and inject bytecode that enables TA/TP on these classes. Instrumentation will happen in-place, i.e. the Eclipse generated class files will be modified directly in their respective output locations. The builder will be triggered on both full and incremental builds - if auto building is enabled on your workspace, an incremental build will be triggered whenever you save one or more files.

The selection of instrumented classes can be configured via the plugin's preferences page. Instrumentation events are logged to a dedicated log view.
