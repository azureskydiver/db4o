Db4oAdmin Usage

This topic applies to .NET version only.

Db4oAdmin is a command line utility. The general syntax is the following:

Usage: Db4oAdmin [options] <assembly>

[options] parameter allows to specify a list of options.

<assembly> parameter allows to pass an assembly, which should be optimized.

Both parameters are optional.

Running Db4oAdmin.exe without any parameters will bring you a short usage hint. This is equivalent to running Db4oAdmin with -? or -help parameter. Additional help information can be retrieved with -help2 or -usage parameters.

The table below gives an explanation of all Db4oAdmin options.

-case-sensitive       

Specifies if optimized queries should be case-sensitive. This option should be used in conjunction with query optimization option (optimize-predicates or cf2-delegates):

Db4oAdmin -optimize-predicates -case-sensitive MyAssembly.exe

-cf2-delegates         

Enable delegate style queries for CompactFramework 2. If you don't use Db4oAdmin with this option on your CF2 assembly, NQ delegates won't be optimized. See Including Db4oAdmin In The Build for more information.

-fake                  

Fake operation mode, assembly won't be written. This option can be used for testing before the actual run.

Db4oAdmin -optimize-predicates -fake MyAssembly.exe

-? -help                     

Show standard help list:

Db4oAdmin -help

-help2                 

Show an additional help list (development use):

Db4oAdmin -help2

-instrumentation:PARAM 

Use custom instrumentation type.

PARAM is a string with a full class definition, like

Db4oAdmin.AbstractAssemblyInstrumentation, Db4oAdmin.exe.

This class must implement Db4oAdmin.IAssemblyInstrumentation interface. To make the creation of a custom instrumentation class easier db4o provides Db4oAdmin.AbstractAssemblyInstrumentation class, which can be used as a template. For an example implementation see Db4oAdmin.TAInstrumentation class.

-optimize-predicates  

Optimize predicate subclasses. This option should be used for build-time NQ optimization:

Db4oAdmin -optimize-predicates MyAssembly.exe

-ta

Instrument classes to support Transparent Activation:

Db4oAdmin -ta MyAssembly.exe

-usage                 

Show usage syntax and exit:

Db4oAdmin -usage

-v -verbose               

Verbose operation mode. Should be combined with the other options:

Db4oAdmin -ta -v MyAssembly.exe

-V -version               

Display version and licensing information:

Db4oAdmin -V

-vv                    

Pretty verbose operation mode:

Db4oAdmin -ta -vv MyAssembly.exe
