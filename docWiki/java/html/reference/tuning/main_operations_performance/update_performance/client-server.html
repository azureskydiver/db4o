Client-Server

Client/server performance is a bit slower than local performance. This is illustrated with the following test:

UpdatePerformanceBenchmark.java: runClientServerTest
01private void runClientServerTest()...{
02      System.out.println("Update test: Client/Server environment");
03      int objectsToUpdate = 30;
04      
05      
06      init();
07      clean();
08      open(configureClientServer());
09      store();
10      System.out.println("Update " + objectsToUpdate + " of " + _count + " objects locally:");
11      updateItems(objectsToUpdate);
12      close();
13      
14      initForClientServer();
15      clean();
16      open(configureClientServer());
17      store();
18      System.out.println("Update " + objectsToUpdate + " of " + _count + " objects remotely:");
19      updateItems(objectsToUpdate);
20      close();
21    }
UpdatePerformanceBenchmark.java: initForClientServer
1private void initForClientServer()...{
2      _count = 1000;
3        _depth = 90;
4        _isClientServer = true;
5        _host = "localhost";  
6    }
UpdatePerformanceBenchmark.java: init
1private void init()...{
2      _count = 1000;
3        _depth = 90;
4        _isClientServer = false;
5          
6    }
UpdatePerformanceBenchmark.java: configureClientServer
1private Configuration configureClientServer()...{
2      Configuration config = Db4o.newConfiguration(); 
3        config.clientServer().singleThreadedClient(true);
4        return config;
5    }

The results:

Update test: Client/Server environment

Store 90000 objects: 7935ms

Update 30 of 1000 objects locally:

Updated 30 items: 404ms

Store 90000 objects: 11421ms

Update 30 of 1000 objects remotely:

Updated 30 items: 436ms

You can see that the performance drop is quite insignificant in this case, however it can be much worse on slow or unreliable networks.
