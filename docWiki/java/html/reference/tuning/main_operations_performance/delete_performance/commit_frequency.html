Commit Frequency

Commit time has a major influence on all db4o operations. Commit is generally a slow operation as it requires physical disk access. The following example shows how the frequency of commits affects the delete performance:

DeletePerformanceBenchmark.java: runCommitTest
01private void runCommitTest()...{
02      System.out.println("Delete test with different commit frequency");
03      
04      configureForCommitTest();
05      initForCommitTest();
06      
07      clean();
08      System.out.println("Test delete all:");
09      open();
10      store();
11      deleteAll();
12      close();
13      
14      
15      
16      clean();
17      System.out.println("Test delete all with commit after each " + _commitInterval + " objects:");
18      open();
19      store();
20      deleteAllWithCommit();
21      close();
22      
23      
24    }

The following initial setup is used:

DeletePerformanceBenchmark.java: initForCommitTest
1private void initForCommitTest()...{
2      _count = 10000;
3      _commitInterval = 1000;
4        _depth = 3;
5        _isClientServer = false;
6    }

Delete procedures are as follows:

DeletePerformanceBenchmark.java: deleteAll
01private void deleteAll() ...{
02    ObjectSet result = objectContainer.queryByExample(null);
03      ArrayList<Long> ids = new ArrayList<Long>();
04      for (Object obj: result)...{
05        ids.add(objectContainer.ext().getID(obj));
06      }
07      startTimer();
08      for (long id: ids)...{
09        objectContainer.delete(objectContainer.ext().getByID(id));
10      }
11      objectContainer.commit();
12      stopTimer("Deleted all objects");
13  }
DeletePerformanceBenchmark.java: deleteAllWithCommit
01private void deleteAllWithCommit() ...{
02      ObjectSet result = objectContainer.queryByExample(null);
03      ArrayList<Long> ids = new ArrayList<Long>();
04      for (Object obj: result)...{
05        ids.add(objectContainer.ext().getID(obj));
06      }
07      startTimer();
08      int i = 0;
09      for (long id: ids)...{
10        objectContainer.delete(objectContainer.ext().getByID(id));
11        if (i++ > _commitInterval)...{
12          objectContainer.commit();
13          i = 0;
14        }
15      }
16      objectContainer.commit();
17      stopTimer("Deleted all objects");
18  }

The results on the test computer are:

Delete test with different commit frequency

Test delete all:

Store 30000 objects: 2432ms

Deleted all objects: 3731ms

Test delete all with commit after each 1000 objects:

Store 30000 objects: 2103ms

Deleted all objects: 8615ms

You can see that the time requires for deletion increased with the amount of commits. To improve the performance commit frequency should be kept relatively low,  in the same time it must ensure the integrity of the logical transactions.
