Reusing MemoryBins

In the previous example we saw how a MemoryBin can be stored to a normal disk file. You can also transfer it to the other parts of an application as a normal variable (byte array).

MemoryStorageExample.java: reuseMemoryBin
01public static void reuseMemoryBin() ...{
02    // Create original storage
03    MemoryStorage origStorage = new MemoryStorage();
04    EmbeddedConfiguration origConfig = Db4oEmbedded.newConfiguration();
05    origConfig.file().storage(origStorage);
06    // A new MemoryBin identified by DB4O_URI will be created
07    ObjectContainer origDb = Db4oEmbedded.openFile(origConfig, DB4O_URI);
08    origDb.store(new Book("Alice in Wonderland"));
09    origDb.close();
10
11    // Retrieve original bin by its URI
12    MemoryBin origBin = origStorage.bin(DB4O_URI);
13    // This bin contains the database data
14    byte[] data = origBin.data();
15
16    // Create a new bin, using the data from the original bin
17    MemoryBin newBin = new MemoryBin(data, new DoublingGrowthStrategy());
18    MemoryStorage newStorage = new MemoryStorage();
19    // Assign the new bin to a new MemoryStorage using the same DB4O_URI
20    newStorage.bin(DB4O_URI, newBin);
21    EmbeddedConfiguration newConfig = Db4oEmbedded.newConfiguration();
22    newConfig.file().storage(origStorage);
23    // Open object container using the new bin with the original data
24    ObjectContainer newDb = Db4oEmbedded.openFile(newConfig, DB4O_URI);
25    // test that the data is still there
26    Book book = (Book) newDb
27        .queryByExample(new Book("Alice in Wonderland")).get(0);
28    System.out.println(book);
29    newDb.close();
30  }

From this and previous example, you can see that MemoryBin API gives you lots of flexibility: combining persistence and exchange strategies you can achieve more complex functionalities, such as synchronization, archiving, versioning, disconnected remote mode etc.
