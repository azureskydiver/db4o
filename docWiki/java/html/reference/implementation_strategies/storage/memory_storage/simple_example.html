Simple Example

Let's have a look at a simple example. For reuse, it is convenient to create a separate method, which will produce the necessary configuration:

MemoryStorageExample.java: getInMemoryConfig
1private static EmbeddedConfiguration getInMemoryConfig() ...{
2    MemoryStorage ms = new MemoryStorage();
3    EmbeddedConfiguration config = Db4oEmbedded.newConfiguration();
4    config.file().storage(ms);
5    return config;
6  }

In this method we use create a default MemoryStorage with doubling growth strategy and a new bin.

From the following code you can see that further usage is the same as with the default disk storage:

MemoryStorageExample.java: createDatabase
01private static void createDatabase(EmbeddedConfiguration config) ...{
02    // the following line is only necessary for the database on disk
03    new File(DB4O_URI).delete();
04
05    long startTime = System.currentTimeMillis();
06    // DB4O_URI will be the name of the database file for the default
07    // configuration and the name of a MemoryBin for in-memory database
08    ObjectContainer container = Db4oEmbedded.openFile(config, DB4O_URI);
09    try ...{
10      Library lib = new Library();
11      for (int i = 0; i < OBJECT_COUNT; i++) ...{
12        lib.addBook(new Book("title" + i, lib));
13      }
14      container.store(lib);
15      Book book = (Book) container.queryByExample(
16          new Book("title1", null)).get(0);
17      System.out.println(book);
18    } finally ...{
19      container.close();
20    }
21    System.out.println(String.format("Time to create a database: %d ms",
22        System.currentTimeMillis() - startTime));
23  }

To demonstrate the performance improvement, please run the method above with the default configuration:
