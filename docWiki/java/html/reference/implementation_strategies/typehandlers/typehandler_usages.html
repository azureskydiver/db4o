This chapter provides some of the typehandler examples, which can be used as a replacement of translators.

Selective Field Persistence

In the first example we will look at the case, when we want to save only certain fields of a class. Suppose we have the following class:

NotStorable.java
01/**//* Copyright (C) 2004 - 2007 db4objects Inc. http://www.db4o.com */
02package com.db4odoc.typehandler.translators;
03
04public class NotStorable ...{
05  private int id;
06
07  private String name;
08
09  private transient int length;
10
11  public NotStorable(int id, String name) ...{
12    this.id = id;
13    this.name = name;
14    this.length = name.length();
15  }
16
17  public int getId() ...{
18    return id;
19  }
20
21  public String getName() ...{
22    return name;
23  }
24
25  public int getLength() ...{
26    return length;
27  }
28
29  public String toString() ...{
30    return id + "/" + name + ": " + length;
31  }
32}

In this case we obviously do not want to store transient length field.  To achieve this we can write a new typehandler, which will store a duplicate of the class without the length field. To make the work easier, we will extend the default FirstClassObjectHandler.  First class means any persistent object that has its identity in the database.

NotStorableTypehandler.java
01package com.db4odoc.typehandler.translators;
02
03import com.db4o.internal.handlers.*;
04import com.db4o.internal.marshall.*;
05import com.db4o.marshall.*;
06import com.db4o.typehandlers.*;
07
08public class NotStorableTypehandler extends FirstClassObjectHandler implements TypeHandler4, FirstClassHandler,
09VariableLengthTypeHandler
10...{
11
12  private class Storable
13  ...{
14    private int id;
15    private String name;  
16  }
17  
18  public void write(WriteContext context, Object obj) ...{
19    NotStorable item = (NotStorable)obj;
20    Storable storableItem = new Storable();
21    storableItem.id = item.getId();
22    storableItem.name = item.getName();
23    
24    context.writeObject(storableItem);
25    }
26
27    public Object read(ReadContext context) ...{
28      UnmarshallingContext _context = (UnmarshallingContext) context;
29      NotStorable item = (NotStorable)_context.persistentObject();
30      Storable storableItem = (Storable)context.readObject();
31        item = new NotStorable(storableItem.id, storableItem.name);
32        return item;
33    }
34
35}

You can see that in this case, we are just replacing NotStorable class with a duplicate class Storable and save it instead.

01private static void tryStoreAndRetrieve(Configuration configuration) ...{

04    try ...{

06      System.out.println("ORIGINAL: " + notStorable);
07      container.store(notStorable);
08    } catch (Exception exc) ...{
09      System.out.println(exc.toString());
10      return;
11    } finally ...{
12      container.close();
13    }
14    container = Db4o.openFile(configuration, DB4O_FILE_NAME);
15    try ...{
16      ObjectSet result = container.queryByExample(NotStorable.class);
17      while (result.hasNext()) ...{
18        NotStorable notStorable = (NotStorable) result.next();
19        System.out.println("RETRIEVED: " + notStorable);
20      }
21    } finally ...{
22      container.close();
23    }
24  }

This works. However, this example has certain limitations:  the fields were in fact discarded and encapsulated into a Storable object. This means that we won't be able to query for fields or index on fields.

Ignore Fields Persistence

In the next example we will use one of the db4o internal typehandlers, which allows to skip all the field information for certain objects. This can be useful to safe the database space when the object information is not required for future retrieval. Let's use the following classes:

PersistentItem.java
01package com.db4odoc.typehandler.translators;
02
03public class PersistentItem ...{
04  String id;
05  TransientItem item;
06  
07  public PersistentItem (String id, TransientItem item)...{
08    this.id = id;
09    this.item = item;
10  }
11  
12  public String toString()...{
13    return id + ": " + item.toString();
14  }
15}
TransientItem.java
01package com.db4odoc.typehandler.translators;
02
03public class TransientItem ...{
04  int id;
05  
06  public TransientItem(int id)...{
07    this.id = id;
08  }
09  
10  public String toString()...{
11    return String.valueOf(id);
12  }
13}

Let's assume that TransientItem field information is not required to be stored in our application. We can save the database space then, by using IgnoreFieldsTypeHandler. The configuration will look like this:

TranslatorExample.java: ignoreFieldsConfiguration
01private static Configuration ignoreFieldsConfiguration() ...{
02    Configuration configuration = Db4o.newConfiguration();
03    // add a custom typehandler support
04        TypeHandlerPredicate predicate = new TypeHandlerPredicate() ...{
05            public boolean match(ReflectClass classReflector) ...{
06              GenericReflector reflector = new GenericReflector(
07                  null, new JdkReflector(Thread.currentThread().getContextClassLoader()));
08          ReflectClass claxx = reflector.forName(TransientItem.class.getName()); 
09          boolean res = claxx.equals(classReflector);  
10                return res;
11            }
12        };
13        
14        configuration.registerTypeHandler(predicate, new IgnoreFieldsTypeHandler());
15        return configuration; 
16  }

We can check the result with the following code:

TranslatorExample.java: ignoreFieldsTypehandlerExample
01private static void ignoreFieldsTypehandlerExample() ...{
02    new File(DB4O_FILE_NAME).delete();
03    Configuration configuration = ignoreFieldsConfiguration();
04    ObjectContainer container = Db4o.openFile(configuration, DB4O_FILE_NAME);
05    try ...{
06      PersistentItem persistentItem = new PersistentItem("1", new TransientItem(11));
07      System.out.println("ORIGINAL: " + persistentItem);
08      container.store(persistentItem);
09    } catch (Exception exc) ...{
10      System.out.println(exc.toString());
11      return;
12    } finally ...{
13      container.close();
14    }
15    container = Db4o.openFile(configuration, DB4O_FILE_NAME);
16    try ...{
17      Query query = container.query();
18      query.constrain(PersistentItem.class);
19      ObjectSet result = query.execute();
20      while (result.hasNext()) ...{
21        PersistentItem persistentItem = (PersistentItem) result.next();
22        System.out.println("RETRIEVED: " + persistentItem);
23      }
24    } finally ...{
25      container.close();
26    }
27  }
