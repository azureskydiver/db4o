Cascaded Behavior

You can use different object events to initiate cascaded behavior on update, activate, delete. The following example shows how to ensure that all the referenced objects are deleted when the parent object is deleted:

CallbacksExample.java: testCascadedDelete
01private static void testCascadedDelete() ...{
02    fillDB();
03    final ObjectContainer container = Db4oEmbedded.openFile(DB4O_FILE_NAME);
04    try ...{
05      // check the contents of the database
06      List<Object> result = container.queryByExample(null);
07      listResult(result);
08
09      EventRegistry registry = EventRegistryFactory
10          .forObjectContainer(container);
11      // register an event handler, which will delete the pilot when his
12      // car is deleted
13      registry.deleted().addListener(new EventListener4() ...{
14        public void onEvent(Event4 e, EventArgs args) ...{
15          ObjectEventArgs queryArgs = ((ObjectEventArgs) args);
16          Object obj = queryArgs.object();
17          if (obj instanceof Car) ...{
18            container.delete(((Car) obj).getPilot());
19          }
20        }
21      });
22      // delete all the cars
23      List<Car> cars = container.query(Car.class);
24      for (Car c: cars) ...{
25        container.delete(c);
26      }
27      // check if the database is empty
28      result = container.queryByExample(null);
29      listResult(result);
30    } finally ...{
31      container.close();
32    }
33  }

Note, that in client/server mode deleted event is only raised on the server side, therefore the code above won't work.
