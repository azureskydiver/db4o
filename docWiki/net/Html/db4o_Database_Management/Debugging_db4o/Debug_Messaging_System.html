Debug Messaging System

 

Db4o messaging system is a special tool, which makes db4o functionality more transparent to the user. It can be used:
-    in debugging session - to find out where the problem can reside;
-    for learning - to watch, what does db4o actually do with the objects.

In order to activate messaging before opening a database file use:

.NET: Db4oFactory.Configure().MessageLevel(level)

where level can be:
level = 0: no messages;
level > 0: normal messages;
level > 1: state messages (new object, object update, delete);
level > 2: activation messages (object activated, deactivated).

In order to set up a convenient output stream for the messages, call:


.NET: Db4oFactory.Configure().SetOut(outStream)


outStream = System.out will set up your system to print messages to console.

#messageLevel(level) also can be set after a database has been opened:


.NET: IObjectContainer#Ext().Configure().MessageLevel(level)

The same applies for #setOut().

Let's use the simplest example to see all types of debug messages:

DebugExample.cs: SetCars
01public static void SetCars()
02    ...{
03      Db4oFactory.Configure().MessageLevel(3);
04      File.Delete(YapFileName);
05      IObjectContainer db=Db4oFactory.OpenFile(YapFileName);
06      try 
07      ...{
08        Car car1 = new Car("BMW");
09        db.Set(car1);
10        Car car2 = new Car("Ferrari");
11        db.Set(car2);
12        db.Deactivate(car1,2);
13        IQuery query = db.Query();
14        query.Constrain(typeof(Car));
15        IObjectSet results = query.Execute();
16        ListResult(results);
17      } 
18      finally 
19      ...{
20        db.Close();
21      }
22      Db4oFactory.Configure().MessageLevel(0);
23    }
DebugExample.vb: SetCars
01Public Shared Sub SetCars()Sub SetCars()
02            Db4oFactory.Configure().MessageLevel(3)
03            File.Delete(YapFileName)
04            Dim db As IObjectContainer = Db4oFactory.OpenFile(YapFileName)
05            Try
06                Dim car1 As Car = New Car("BMW")
07                db.Set(car1)
08                Dim car2 As Car = New Car("Ferrari")
09                db.Set(car2)
10                db.Deactivate(car1, 2)
11                Dim query As IQuery = db.Query()
12                query.Constrain(GetType(Car))
13                Dim results As IObjectSet = query.Execute()
14                ListResult(results)
15            Finally
16                db.Close()
17            End Try
18            Db4oFactory.Configure().MessageLevel(0)
19        End Sub

Output looks quite messy, but allows you to follow the whole process. For debugging purposes messaging system provides a timestamp and internal ID information for each object (first number in state and activate messages).
