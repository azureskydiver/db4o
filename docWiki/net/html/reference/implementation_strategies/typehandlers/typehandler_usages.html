This chapter provides some of the typehandler examples, which can be used as a replacement of translators.

Selective Field Persistence

In the first example we will look at the case, when we want to save only certain fields of a class. Suppose we have the following class:

15        }

In this case we obviously do not want to store transient length field.  To achieve this we can write a new typehandler, which will store a duplicate of the class without the length field. To make the work easier, we will extend the default FirstClassObjectHandler.  First class means any persistent object that has its identity in the database.

CultureInfoTypehandler.CS
01/**//* Copyright (C) 2004 - 2007 db4objects Inc. http://www.db4o.com */
02using System.Globalization;
03
04using Db4objects.Db4o.Internal.Handlers;
05using Db4objects.Db4o.Internal.Marshall;
06using Db4objects.Db4o.Marshall;
07using Db4objects.Db4o.Typehandlers;
08
09
10namespace Db4objects.Db4odoc.Typehandler.Translators
11...{
12    public class CultureInfoTypehandler : FirstClassObjectHandler, ITypeHandler4, IFirstClassHandler, IVariableLengthTypeHandler
13    ...{
14        private class Storable
15        ...{
16            private string[] items;
17
18            public Storable(string[] items)
19            ...{
20                this.Items = items;
21            }
22
23            public string[] Items
24            ...{
25                get ...{ return items; }
26                set ...{ items = value; }
27            }
28        }
29
30        public void Write(IWriteContext context, object obj)
31        ...{
32            LocalizedItemList list = (LocalizedItemList)obj;
33            Storable storable = new Storable(list.Items);
34
35            context.WriteObject(storable);
36        }
37
38        public object Read(IReadContext context)
39        ...{
40            UnmarshallingContext _context = (UnmarshallingContext)context;
41            LocalizedItemList list = (LocalizedItemList) _context.PersistentObject();
42            Storable storable = (Storable)context.ReadObject();
43            list = new LocalizedItemList(CultureInfo.CurrentCulture, storable.Items);
44            return list;
45        }
46
47    }
48}

You can see that in this case, we are just replacing NotStorable class with a duplicate class Storable and save it instead.

This works. However, this example has certain limitations:  the fields were in fact discarded and encapsulated into a Storable object. This means that we won't be able to query for fields or index on fields.

Ignore Fields Persistence

In the next example we will use one of the db4o internal typehandlers, which allows to skip all the field information for certain objects. This can be useful to safe the database space when the object information is not required for future retrieval. Let's use the following classes:

PersistentItem.cs
01using System;
02using System.Collections.Generic;
03using System.Text;
04
05namespace Db4objects.Db4odoc.Typehandler.Translators
06...{
07    public class PersistentItem
08    ...{
09        string id;
10        TransientItem item;
11
12        public PersistentItem(string id, TransientItem item)
13        ...{
14            this.id = id;
15            this.item = item;
16        }
17
18        public override String ToString()
19        ...{
20            return id + ": " + item.ToString();
21        }
22    }
23}

Let's assume that TransientItem field information is not required to be stored in our application. We can save the database space then, by using IgnoreFieldsTypeHandler. The configuration will look like this:

TranslatorExample.cs: IgnoreFieldsConfiguration
1private static IConfiguration IgnoreFieldsConfiguration()
2        ...{
3            IConfiguration configuration = Db4oFactory.NewConfiguration();
4            // add a custom typehandler support
5            ITypeHandlerPredicate predicate = new TransientHandlerPredicate();
6
7            configuration.RegisterTypeHandler(predicate, new IgnoreFieldsTypeHandler());
8            return configuration;
9        }
