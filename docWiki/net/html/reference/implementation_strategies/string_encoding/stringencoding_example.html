StringEncoding Example

Lets look at an example string encoding implementation. We need to provide 2 methods: encode, which takes a string and produces byte array and decode, taking a byte array and producing a string. To make it interesting, we will "encrypt" the bytes stored in the database, by shifting them 1 byte ahead with the following class:

01Private Class StringEncodingEncryptedClass StringEncodingEncrypted
02            Implements IStringEncoding
03
04            Public Function Encode()Function Encode(ByVal str As String) As Byte() Implements IStringEncoding.Encode
05                str = Encryption.Encrypt(str)
06                Dim length As Integer = str.Length
07                Dim chars As Char() = New Char(length) {}
08                chars = str.ToCharArray(0, length)
09                Dim bytes As Byte() = New Byte(length * 2 - 1) {}
10                Dim count As Integer = 0
11                For i As Integer = 0 To length - 1
12                    bytes(count) = CByte(Microsoft.VisualBasic.AscW(chars(i)) And &HFF)
13                    count = count + 1
14                    bytes(count) = CByte(Microsoft.VisualBasic.AscW(chars(i)) >> 8)
15                    count = count + 1
16                Next
17                Return bytes
18            End Function
19
20            Public Function Decode()Function Decode(ByVal bytes As Byte(), ByVal start As Integer, ByVal length As Integer) As String Implements IStringEncoding.Decode
21                Dim stringLength As Integer = length / 2
22                Dim chars As Char() = New Char(stringLength) {}
23                Dim j As Integer = start
24                For ii As Integer = 0 To stringLength - 1
25                    chars(ii) = Microsoft.VisualBasic.ChrW(((bytes(j) And &HFF) Or ((bytes(j + 1) And &HFF) << 8)))
26                    j = j + 2
27                Next
28                Return Encryption.Decrypt(New String(chars, 0, stringLength))
29            End Function
30        End Class

You can see that any string code can be easily integrated into the encoding procedure.

In order to test that the code works it is actually enough to just open the database 2 times: as it was mentioned before, internal db4o metadata is stored as strings and therefore opening the database second time will fail if the StringEncoding is not correct.
