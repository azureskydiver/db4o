Let's check how different types of queries work:

EnumExample.cs: RunSODAQuery
1private static IObjectSet RunSODAQuery(IObjectContainer db)
2        ...{
3            IQuery query = db.Query();
4            query.Constrain(typeof(Door));
5            query.Descend("_state").Constrain(DoorState.Open);
6
7            return query.Execute();
8        }
EnumExample.cs: RunLinqQuery
1private static IEnumerable RunLinqQuery(IObjectContainer db)
2        ...{
3            return db.Query((Door d) => d._state == DoorState.Open);
4        }
EnumExample.cs: RunQueryByExample
1public static IObjectSet RunQueryByExample(IObjectContainer db)
2        ...{
3            return db.QueryByExample(new Door(DoorState.Open));
4        }
EnumExample.vb: RunSODAQuery
1Private Shared Function RunSODAQuery()Function RunSODAQuery(ByVal db As IObjectContainer) As IObjectSet
2            Dim query As IQuery = db.Query()
3            query.Constrain(GetType(Door))
4            query.Descend("_state").Constrain(DoorState.Open)
5
6            Return query.Execute()
7        End Function
EnumExample.vb: RunLinqQuery
1Private Shared Function RunLinqQuery()Function RunLinqQuery(ByVal db As IObjectContainer) As IEnumerable
2            Return db.Query(Function (()Function(d As Door) d._state = DoorState.Open)
3        End Function
EnumExample.vb: RunQueryByExample
1Public Shared Function RunQueryByExample()Function RunQueryByExample(ByVal db As IObjectContainer) As IObjectSet
2            Return db.QueryByExample(New Door(DoorState.Open))
3        End Function

You can see that SODA and LINQ queries return the correct results. Query By Example has the limitations with the 0 value as it is considered to be a null template and returns all values.

Let's see how many enum objects did we save in the database:

EnumExample.cs: GetAllStates
1private static int GetAllStates(IObjectContainer db)
2        ...{
3            return db.Query(typeof(DoorState)).Count;
4        }
EnumExample.vb: GetAllStates
1Private Shared Function GetAllStates()Function GetAllStates(ByVal db As IObjectContainer) As Integer
2            Return db.Query(GetType(DoorState)).Count
3        End Function

Well, it's zero - which means that enums don't have database identity and are only stored as part of their parent object.
