TypeAlias

TypeAlias constructor accepts 2 parameters:

  • stored class name - storedType
  • runtime class name - runtimeType

Note that the runtimeType class should exist in your application when you configure the alias.

The alias matches are found by comparing full names of the stored and runtime classes

Let's declare a new TypeAlias

c#:

private static TypeAlias tAlias;

VB:

Private Shared tAlias As TypeAlias

The following method will initialize tAlias and configure db4o to use it:

AliasExample.cs: ConfigureClassAlias
01private static IEmbeddedConfiguration ConfigureClassAlias()
02        ...{
03            // create a new alias
04            tAlias = new TypeAlias("Db4odoc.Aliases.Pilot, Db4odoc", "Db4odoc.Aliases.Driver, Db4odoc");
05            // add the alias to the db4o configuration 
06            IEmbeddedConfiguration configuration = Db4oEmbedded.NewConfiguration();
07            configuration.Common.AddAlias(tAlias);
08            // check how does the alias resolve
09            Console.WriteLine("Stored name for Db4odoc.Aliases.Driver: " + tAlias.ResolveRuntimeName("Db4odoc.Aliases.Driver, Db4odoc"));
10            Console.WriteLine("Runtime name for Db4odoc.Aliases.Pilot: " + tAlias.ResolveStoredName("Db4odoc.Aliases.Pilot, Db4odoc"));
11            return configuration;
12        }

We can always check the results of adding an alias using resolveRuntimeName and resolveStoredName as you see in the example. Basically the same methods are used internally by db4o to resolve aliased names.

06            ...{
