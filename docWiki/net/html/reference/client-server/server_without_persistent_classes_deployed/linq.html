LINQ

Db4o LINQ is an optimisable query: under the hood it is converted to SODA where possible. However, SODA equivalents may not exist for some more complex LINQ (like aggregates). In this case the query will run against all object instances to return the correct result. In case when class definitions are not available - this is not possible.

An example of LINQ query that won't work when class definitions are not available on the client side:

Client.vb: LinqGroupByQuery
01Private Shared Sub LinqGroupByQuery()Sub LinqGroupByQuery()
02            Dim container As IObjectContainer = Database()
03            If container IsNot Nothing Then
04                Try
05                    Dim result = From p As Pilot In container _
06                                  Order By p.Points Descending _
07                                  Select p Group By p.Points, p.Name Into Group Select Name, Points
