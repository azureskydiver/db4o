Configuration

db4o provides a wide range of configuration options to help you meet your performance and reliability requirements. The following example shows how different configurations affect update performance:

UpdatePerformanceBenchmark.cs: Configure
1private IConfiguration Configure()
2        ...{
3            IConfiguration config = Db4oFactory.NewConfiguration();
4            // using MemoryIoAdapter improves the performance 
5            // by replacing the costly disk IO operations with 
6            // memory access
7            config.Io(new MemoryIoAdapter());
8            return config;
9        }
UpdatePerformanceBenchmark.cs: ConfigureCascade
1private IConfiguration ConfigureCascade()
2        ...{
3            IConfiguration config = Db4oFactory.NewConfiguration();
4            // CascadeOnUpdate can be a performance-killer for 
5            // deep object hierarchies
6            config.ObjectClass(typeof(Item)).CascadeOnUpdate(true);
7            return config;
8        }
UpdatePerformanceBenchmark.cs: ConfigureTP
1private IConfiguration ConfigureTP()
2        ...{
3            IConfiguration config = Db4oFactory.NewConfiguration();
4            // With Transparent Persistence enabled only modified
5            // objects are written to disk. This allows to achieve 
6            // better performance
7            config.ObjectClass(typeof(Item)).CascadeOnUpdate(true);
8            return config;
9        }
