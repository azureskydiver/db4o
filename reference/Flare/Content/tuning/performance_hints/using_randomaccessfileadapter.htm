Using FileStorage

You can use the FileStorage directly, avoiding the default caching strategy. Take a look you can configure the storage. See "FileStorage"

 

Advantage

Uses less memory than default CachingStorage.

Effect

By default db4o uses CachingStorage, which allows to improve bulk write performance, by creating cached pages in memory and thus reducing the amount of physical disc writes. This of course can increase the memory consumption, which can be a critical resource in some cases. To free up some additional memory it might be reasonable to switch to FileStorage.

Alternate Strategies

The default CachingStorage and configurable MemoryStorage allow to achieve better performance by utilizing RAM instead of hard-drive.
