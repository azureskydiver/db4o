Optimization at Query Execution Time

Note: This will not work with JDK1.1.

To enable code analysis and optimization of native query expressions at query query execution time. Optimization can be turned on and off with the following configuration setting:

configuration.common().optimizeNativeQueries(boolean optimizeNQ)

configuration.Common.OptimizeNativeQueries = optimizeNQ

Ensure that you have the db4o--nqopt.jar and bloat-1.x.jar in your classpath. Otherwise the runtime optimization wont't work.

Ensure that you've added the Db4objects.Db4o.NativeQueries.dll, Mono.Cecil.dll and Cecil.FlowAnalysis.dll assembly to your project. Otherwise the runtime optimisation won't work.
