Freespace Filler

FreespaceFiller/IFreespaceFiller can be used to rewrite deleted objects information in the database file. When an object gets deleted, normally it is just marked as deleted, until the next defragment, i.e. the full object information can still be read. In some cases this behaviour can be not desirable. One of the options to permanently remove the object's information from the database file is obviously defragment. However, it is not possible to run defragment after each object's deletion. Another option is to implement FreespaceFiller/IFreespaceFiller interface and enable it in the configuration:

Java:

config.file().freespace().freespaceFiller(CustomFreespaceFiller);

.NET:

config.File().Freespace().FreespaceFiller(CustomFreespaceFiller);

Note, that using freespace filler slows down IO operations.
