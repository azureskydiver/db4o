Automatic Properties

Automatic properties were introduced in c# .NET 3.5 version. Effectively automatic properties simply allow a user to skip the step of introducing private field variables when the property simply sets and gets the field value, i.e:

class Car{ public string Model {get; set}}

Is equal to:

class Car{private string _model;public string Model { get { return _model; }; set { _model = value; }}

As you can see automatic properties version allows to save quite a bit of code, which can be very valuable if your classes contain lots of properties. The benefit over using public fields is the same as with normal properties - easy refactoring and adding functionality into the property code.

Note, that automatic properties feature is not available in VB.NET.
