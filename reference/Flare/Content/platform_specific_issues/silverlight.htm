Silverlight

db4o runs with some limitations on Microsoft's Silverlight. You can find the Silverlight assemblies in the db4o distribution, in the folder 'bin/silverlight'.

Feature Set

Currently on a very limited feature set is supported. Right now only the db4o core is supported. This means that there's no LINQ-support and no client server mode.

Isolated Storage

By default you cannot access the file system from Silverlight, but only a special isolated storage. Therefore you need to use the isolated storage system also for db4o. Note that by default this storage is limited to 1 MByte of data. You can increase it with the users consent.

Persistent Object Limitations

Due to the restrictions of the Silverlight sandbox, it's impossible to inspect the private fields of an object. This means that the field of persistent objects need to be public!

Query Limitations

As for now, the Silverlight port of db4o doesn't support LINQ. You can use native queries, but those are not optimized on the Silverlight platform and therefore slow. The only efficient query-method for now it using SODA directly.
