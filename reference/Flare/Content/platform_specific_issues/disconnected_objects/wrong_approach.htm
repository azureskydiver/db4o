Wrong Approach

The wrong approach is to try to store disconnected objects. Since db4o manages object by their object-identity and doesn't recognize objects which have been serialized or loaded by another object container instance. This example shows, that instead of updating the object, db4o will store a new instance of the object. Because the object is loaded and stored in different object container.

So in order to update a object, you need to load and store it in the same object-container. If you cannot do this, you need to merge to object-changes. See "Example Merge Changes"
