Merging Changes

Merging the changes is the most challenging part when using disconnected objects. Imagine that we've a disconnected object, which contains the changes. We have to store the changes somehow. You cannot simply store the disconnected objects, because db4o wouldn't recognize it and store a new object instead of updating the old object. See "Wrong Approach"

Instead you need to load the existing object and then copy the state from the disconnected object to the loaded object. Basically you traverse the object-graph and copy all changes over. See "Example Merge Changes"

db4o has no built-in merge support. However there are external libraries which can help you to merge changes, like Automapper Dozer.
