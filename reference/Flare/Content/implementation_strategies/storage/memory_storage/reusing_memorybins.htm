Reusing MemoryBins

In the previous example we saw how a MemoryBin can be stored to a normal disk file. You can also transfer it to the other parts of an application as a normal variable (byte array).

From this and previous example, you can see that MemoryBin API gives you lots of flexibility: combining persistence and exchange strategies you can achieve more complex functionalities, such as synchronization, archiving, versioning, disconnected remote mode etc.
