Viewing Enum Values In OME


As it was mentioned before enums do not carry any database identity and are stored in their parent object slot. If you open a database with Enum values in OME, you will be able to see the enum class in the db4o Browser view, however if you proceed and select "Show all objects" from the right-click menu - you will see that there are no Enum objects stored in the database.

But don't worry - Enum is just an integer, so all we actually need is just values of the Enum fields in the database.  Please, select a class with the Enum field in the db4o Browser view (if you are using the database from the previous example - Door class). Right-click it and select "Show all objects".

Something went wrong here - there are no Enum values and nulls are stored instead. Is the information lost?

Here you will need to understand how Enum values are stored in the database. In order to save space in the database file, db4o stores only the absolutely necessary information for the enums:

  • Enum class id
  • Enum numeric value

The actual Enum representation will be picked up in the runtime using Enum class definition. As OME by default runs without class definitions using Generic Reflector, OME can't interprete the Enum field value correctly.  In order to fix it go to "Tools" -> "Object Manager Enterprise" -> "Options" menu and select "Assembly Search Path..." and add the path to the assembly, containing the Enum definition.
