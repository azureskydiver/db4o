WildcardAlias

WilldcardAlias allows creating aliases for packages, namespaces or multiple similar classes. WilcardAlias constructor accepts 2 parameters:

  • storedPattern
  • runtimePattern

* symbol is used to specify the place where multiple matches are allowed (you can use only one * per pattern).

Let's look how to alias all classes within one package/namespace.

AliasExample.java: savePilots
01private static void savePilots() ...{
02    new File(DB4O_FILE_NAME).delete();
03    ObjectContainer container = Db4oEmbedded.openFile(Db4oEmbedded
04        .newConfiguration(), DB4O_FILE_NAME);
05    try ...{
06      Pilot pilot = new Pilot("David Barrichello", 99);
07      container.store(pilot);
08      pilot = new Pilot("Kimi Raikkonen", 100);
09      container.store(pilot);
10    } finally ...{
11      container.close();
12    }
13  }
AliasExample.java: configureAlias
01private static EmbeddedConfiguration configureAlias() ...{
02    // com.db4odoc.aliases.* - package for the classes saved in the database
03    // com.db4odoc.aliases.newalias.* - runtime package
04    WildcardAlias wAlias = new WildcardAlias("com.db4odoc.aliases.*",
05        "com.db4odoc.aliases.newalias.*");
06    // add the alias to the configuration
07    EmbeddedConfiguration configuration = Db4oEmbedded.newConfiguration();
08    configuration.common().addAlias(wAlias);
09    System.out
10        .println("Stored name for com.db4odoc.aliases.newalias.Pilot: "
11            + wAlias
12                .resolveRuntimeName("com.db4odoc.aliases.newalias.Pilot"));
13    System.out.println("Runtime name for com.db4odoc.aliases.Pilot: "
14        + wAlias.resolveStoredName("com.db4odoc.aliases.Pilot"));
15    return configuration;
16  }

AliasExample.cs: ConfigureAlias
01private static IEmbeddedConfiguration ConfigureAlias()
02        ...{
03            // Db4odoc.Aliases.* - namespace for the classes saved in the database
04            // Db4odoc.Aliases.NewAlias.* - runtime namespace
05            WildcardAlias wAlias = new WildcardAlias("Db4odoc.Aliases.*", "Db4odoc.Aliases.NewAlias.*");
06            // add the alias to the configuration
07            IEmbeddedConfiguration configuration = Db4oEmbedded.NewConfiguration();
08            configuration.Common.AddAlias(wAlias);
09            Console.WriteLine("Stored name for Db4odoc.Aliases.NewAlias.Pilot: " + wAlias.ResolveRuntimeName("Db4odoc.Aliases.NewAlias.Pilot"));
10            Console.WriteLine("Runtime name for Db4odoc.Aliases.Pilot: " + wAlias.ResolveStoredName("Db4odoc.Aliases.Pilot"));
11            return configuration;
12        }

You can check the matches for the concrete classes using resolveRuntimeName and resolveStoredName.

In order to add your own aliasing logic implement Alias interface (for example you may want to use more sophisticated pattern logic). You own resolving logic implementation should reside in resolveRuntimeName and resolveStoredName methods.
