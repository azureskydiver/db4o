Transparent Persistence For Java Collections

One of the most valuable usages of TA/TP is in collections:

  • Using TA you don't have to worry about activating the next level of objects
  • Using TP you can add new members to the collection or update existing ones without worrying about storing the changes to the object container.

If you remember the TA basics, in order to make an object activatable its class must implement Activatable interface. This can be done easily with the new classes, however in collections case you will probably want to use standard Java collection, which can't be modified. Db4o solves this problem by deriving special activatable classes from the standard Java collections and overriding all the public methods adding activate() call before calling the base method. At present db4o supports all Java5 standard collections:

ActivatableArrayList ActivatableHashMap ActivatableHashSet ActivatableHashtable ActivatableLinkedList ActivatableStack ActivatableTreeSet

In order to make use of these classes you will only need to add TransparentPersistenceSupport to your configuration before opening the object container:

TACollectionsExample.java: openObjectContainer
1private static ObjectContainer openObjectContainer() ...{
2    EmbeddedConfiguration embeddedConfiguration = Db4oEmbedded
3        .newConfiguration();
4    embeddedConfiguration.common().add(new TransparentPersistenceSupport());
5    return Db4oEmbedded.openFile(embeddedConfiguration, DB4O_FILE_NAME);
6  }

And in the end enhance the project adding TP support (example will be shown below).

The following example code shows how Team class can be used:

TACollectionsExample.java: main
01public static void main(String[] args) ...{
02    new File(TACollectionsExample.DB4O_FILE_NAME).delete();
03    ObjectContainer objectContainer = openObjectContainer();
04    Team team = new Team("Ferrari");
05    // Store the team first time to make it persistent
06    objectContainer.store(team);
07    objectContainer.close();
08    
09    objectContainer = openObjectContainer();
10    Team team1 = queryTeamByName(objectContainer, "Ferrari");
11    team1.addPilot(new Pilot("Felipe Massa"));
12    team1.addPilot(new Pilot("Kimi Raikkonen"));
13    // No need to re-store team - it will be stored automatically
14    objectContainer.close();
15    
16    
17    objectContainer = openObjectContainer();
18    // check the results
19    ObjectSet<Team> objectSet = objectContainer.query(Team.class);
20    while (objectSet.hasNext()) ...{
21      System.out.println(objectSet.next());
22    }
23    objectContainer.close();
24  }

Conversion to Activatable db4o collections and TA enhancement will be done by an ant script using Db4oEnhancerAntTask. This task can be integrated in compile and run script as follows:

build.xml
01<project name="db4o.TP.sample" basedir="." default="run">
02
03  <property name="dir.src" value="${basedir}/src" />
04  <property name="dir.bin" value="${basedir}/bin" />
05  
06  <path id="project.classpath">
07      <fileset dir="lib">
08        <include name="**/*.jar"/>
09      </fileset>
10  </path>
11  
12  <taskdef name="db4o-enhance" 
13    classname="com.db4o.enhance.Db4oEnhancerAntTask" 
14    classpathref="project.classpath" />
15  
16  <target name="compile">
17    <mkdir dir="${dir.bin}" />
18    <javac srcdir="${dir.src}" destdir="${dir.bin}" failonerror="true">
19      <classpath refid="project.classpath" />
20    </javac>
21  </target>
22  
23  <target name="enhance" depends="compile">
24    <db4o-enhance classtargetdir="${dir.bin}" ta="true" nq="true" collections="true">
25      <classpath refid="project.classpath" />
26      <sources dir="${dir.bin}">
27        <include name="**/*.class" />
28      </sources>
29    </db4o-enhance>
30  </target>
31
32  <target name="run" depends="enhance">
33    <java classname="do4odoc.tacollections.TACollectionsExample" failonerror="true">
34      <classpath>
35        <pathelement path="${dir.bin}" />
36        <path refid="project.classpath" />
37      </classpath>
38    </java>
39  </target>
40
41</project>

If you will run this script you will see that the new team members were saved automatically when the object container was closed. To see the effect of the enhancer, please try to run TACollectionExample from IDE (without enhancement), in this case standard Java collections will be used and TA/TP will have no effect.
