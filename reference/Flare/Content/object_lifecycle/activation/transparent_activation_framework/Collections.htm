Collections

In order to support properly, a class need to implement the -interface. For your domain-classes this is quite easy to archive. But what about the -collections? Wouldn't it be nice if the collections integrate also with the transparent activation framework?

db4o brings special, transparent activation aware collections with it. This collections load the content only when the collections is actually used.

It's recommended to use the collection-interfaces wherever possible instead of the concrete classes. This avoids unnecessary direct dependencies on the implementations and makes it easy to swap the implementations.

You can use the db4o-collection directly in your code. For example in the case you implement the transparent activation support yourself. Take a looks a few tips: See "Use the TA collections directly"

The enhancement tools can automatically replace the -collections with the db4o-equivalent. However there are few rules and limitations. See "Enhance Collections"
