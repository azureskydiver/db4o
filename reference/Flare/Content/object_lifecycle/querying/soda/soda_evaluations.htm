SODA Evaluations

Sometimes the capabilities of the regular SODA-query is not enough. In such cases you can add evaluations to the SODA-query. A evaluation is a piece of code which runs against the real objects.

To use a evaluation, you need to pass a instance of the -interface as a constrain. Then implement the match-method of the -interface. In the match-method you can get the candidate-object, the object-container and a include-method. Use your constrains on the object. If it matches, pass true to the include-method. Otherwise pass false.

While SODA evaluations are extremely powerful they are also slow. In order to run the evaluation the objects need to be instantiated from the database and then processed by the evaluator. This means that you should use evaluations only when there's no other possibility.
