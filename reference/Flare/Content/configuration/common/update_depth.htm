Update Depth

By default db4o only stores changes on the updated object, but not the changes on referenced objects. With a higher update-depth db4o will traverse along the object graph to a certain depth and update all objects. See "Update Depth".

With the update-depth you configure how deep db4o updates the object-graph.

A higher update depth is usually more convenient to work with. However the higher the update depth the more time it takes to update the objects. Therefore its a tradeoff. Note that you can also use transparent persistence, which takes of updating the right objects.

Class Specific Configuration

You can also configure a class specific update depth. See "ObjectClass Configuration"
