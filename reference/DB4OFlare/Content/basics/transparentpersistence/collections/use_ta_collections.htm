Using Transparent Activatable Collections Directly

You can use the transparent activation aware collections directly in your code. Their behavior is the same as the -collections. Here are a few tips:

  • Prefer the collection-interfaces over the concrete classes in your field-, parameter- and return-type declarations. For example declare a field as a IList<T> instead of a List<T>List<T> instead of an ArrayList<T>
  • Maybe it is useful to create a collection factory in your code. Then you have only one place in your code which decides which implementation is used. That makes it easy to replace the implementations.
