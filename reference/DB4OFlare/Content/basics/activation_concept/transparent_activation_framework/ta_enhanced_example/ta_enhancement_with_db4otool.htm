TA Enhancement With Db4oTool

TA instrumentation can be done by applying Db4oTool utility to the ready .NET assemblies:

Db4oTool -ta -debug YourAssemblyName

Now you probably don't want to call the Db4oTool every time you compile. You can automate that by adding it to the build step. You can do that in Visual Studio by adding the command line to run in the build settings. Go to the project settings of your project. Select 'Build Events' in the project-settings. Then add the command to run the Db4oTool in the 'Post build event command line'. Note that you can use variables to find the right path for your assembly.
