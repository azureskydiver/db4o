Transient Classes

Some of the classes are not supposed to be persistent. Of course you can avoid saving their instances in your code and mark all their occurrences in another classes as transient. But that needs some attention and additional coding. You can achieve the same result in an easier way using TransientClass interface:

Java:

com.db4o.types.TransientClass

C#:

Db4objects.Db4o.Types. ITransientClass

VB:

Db4objects.Db4o.Types. ITransientClass

When a class implements this marker interface, instances of that class are not stored. If a field contains a reference to such an object, it will be null when you load it from the database.
